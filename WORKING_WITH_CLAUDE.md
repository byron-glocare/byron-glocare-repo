# Claude Code 협업 방식 (Working Agreement)

> 이건 핸드오프(작업 인계)가 아니라, **운영자와 Claude Code가 일하는 고정 규칙**이다.
> CLAUDE.md 에서 `@import` 되어 매 세션 자동 로드된다. 세션마다 반복되던 비효율을
> 막기 위한 것이니, 새 채팅에서도 이 규칙을 먼저 따른다.

---

## 1. 로컬 실행·명령은 PowerShell 로 그냥 한다

- 앱/서버 실행, 빌드, 스크립트는 **PowerShell 로 직접** 실행한다. `npm run dev` 는 백그라운드로.
- **브라우저 검증**: 관리형 preview 브라우저는 이 환경에서 자주 **얼어붙는다**
  (레포가 OneDrive 폴더라 `.next` 동기화로 dev 가 매우 느림 — `dev_onedrive_slow_fs` 메모 참조).
  → dev 서버를 PowerShell 로 띄우고 **운영자가 자기 Chrome 에서 `localhost:3000` 확인**하거나,
    인증된 상태에서 `fetch(path)` 로 SSR HTML 을 받아 검증한다.
  → preview 브라우저 네비게이션을 억지로 우회하는 삽질 금지.
- **원칙**: 실행이 막히면 새 방식을 발명하기 전에 "그냥 PowerShell 로 하면 되는가?" 부터 본다.

## 2. Supabase 스키마 변경 = SQL 을 써주면 운영자가 에디터에 붙여넣는다

- DB 변경이 필요하면 Claude 가 **SQL 구문을 작성** → 운영자가 **Supabase 대시보드 SQL 에디터**에
  붙여넣어 실행한다. 이게 이 프로젝트의 표준 방식.
- **제안 금지**: supabase CLI / `db pull` / `link` / DB 비밀번호 요구 / 로컬 Postgres 가정.
  (이 프로젝트는 그런 워크플로를 안 쓴다. 괜히 제안해서 대화 길어지지 말 것.)
- 마이그레이션 `.sql` 파일은 **기록·재현용**으로 `supabase/migrations/` 에 남긴다.
  append-only — 이미 적용된 파일은 수정하지 말고 새 번호로 추가.
- **이미 운영에 존재하는 테이블의 사후 박제 마이그레이션은 운영 DB 에 실행하지 않는다**
  (신규/빈 프로젝트에서만). 멱등으로 써도 RLS 정책 재실행이 운영 보안을 바꿀 수 있음.

## 2-1. 환경 구조: DB 는 단일 라이브 Supabase 공유, 코드만 dev/prod 분리

- **데이터베이스**: 로컬 개발과 운영(Vercel)이 **같은 라이브 Supabase 프로젝트
  (`oczjvsxmlbuicyhheelc`) 하나**를 공유한다. 별도 dev/staging DB 없음.
  → 로컬에서 개발해도 DB 작업은 **라이브 DB 에 직접** 반영된다.
- **코드**: dev(로컬) ↔ prod(Vercel 배포)는 분리. 코드만 "개발 후 배포".
- **안전 규칙 (운영자 확인)**: 라이브 DB 에 **추가/테스트는 안전**하다 —
  운영 코드가 새 테이블/행을 노출하지 않거나, 모든 노출이 `active`/`is_active`/
  `status` 플래그 + RLS 로 게이트되기 때문. **단 기존 데이터를 삭제·파괴하지 말 것.**
  (그래서 자동 생성 레코드는 `active=false` 로 만든다 — 추가되지만 노출 안 됨.)

## 3. 커스텀 서브에이전트를 활용한다 (컨텍스트 절약)

운영자가 만든 user-level 서브에이전트 (`~/.claude/agents/`, 모든 프로젝트 공용):

- **doc-surveyor** — 대용량 문서(PRD·기획서·회의록·마이그레이션 SQL 등)를 메인 컨텍스트에
  통째로 올리지 말고 구조·확정 결정·미해결 TBD·참조 관계만 ≤400 토큰으로 정찰.
- **research-summarizer** — 외부 웹 조사·검색·문서 fetch(가격비교·시장·법무·API 스펙 등)를
  핵심만 ≤250 토큰으로 요약(출처 포함).

→ 큰 문서를 읽거나 외부 조사가 필요하면 **메인 컨텍스트를 아끼려 이 에이전트를 먼저** 쓴다.

## 4. 컨텍스트 절약을 기본값으로

- 이 프로젝트는 1M 컨텍스트 소진으로 세션이 죽은 적이 있다.
- 파일을 통째로 읽기보다 필요한 부분만. 큰 문서는 doc-surveyor. 무거워지면 운영자가 `/compact`.
- 1M 컨텍스트 모드는 끈다(운영자 지시).

## 5. 일반 작업 규칙

- 같은 목적의 우회를 **반복하지 않는다**. 막히면 발명하기 전에 운영자의 기존 방식을 먼저 확인한다.
- 커밋은 운영자가 **명시적으로 요청할 때만** 한다.
- 한국어로 소통한다.
- 도메인 2개(요양보호사 + 유학생)를 공유 파일(`nav.ts`/대시보드/`validators.ts`)에서 건드릴 땐
  양쪽 영향 확인. `validators.ts` 는 append-only.
