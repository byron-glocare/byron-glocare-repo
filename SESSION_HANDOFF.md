## 1. SKU 모델 2개로 확정 (위치 기준)

| | **SKU-1: 베트남 → 한국** | **SKU-2: 한국 D-4 → D-2** |
|---|---|---|
| 대상 학생 위치 | 베트남 거주 | 한국 거주 (D-4 어학당 재학) |
| 내부 옵션 | 어학당 진학 / 본과 직진 (선택) | 본과 전과 (단일) |
| 핵심 서비스 | AI 서류 + 송영 + 현지서류 + 면접코칭 | AI 서류 위주 (송영·현지서류 없음) |
| 가격 수준 | 높음 | 낮음 (별도 설정) |
| 채널 | 학생 직접 + 브로커 어필리에이트 | 학생 직접 + 브로커 어필리에이트 |

**유학원(B2B) 채널 3** = **MVP 제외**. 시스템 등록 없이 매출 합산용 회계 기록만.

## 2. 가격 정책 3층 구조 (모두 어드민 설정)

```
정가 (List Price)
 ↓
- 상시 할인 (Standard Discount) — 학생 직접 결제 시 자동 적용
 ↓
회사 기본 매출 (Company Base)
 ↓
+ 티어 인센티브 (회사 마진 = 정가 - 상시할인 의 일부)
```

브로커 채널:
- 브로커 수수료 = **상시 할인 폭 안에서 자유 배분**
  - 수수료 $0 → 학생은 풀 할인
  - 수수료 = 풀 max → 학생은 정가 결제, 할인폭 전부 브로커가 가져감
- 티어 인센티브는 **별도 추가** (회사 마진에서 차감)

→ **SKU 별로 독립 설정**. 어드민에 가격 정책 화면 필요.

## 3. 브로커-SKU 협상 매트릭스

같은 브로커도 SKU 마다 다른 수수료 협상 가능:

```
브로커 응웬 유학원
├─ SKU-1 (베트남→한국): 수수료 $X, 티어 Y
└─ SKU-2 (한국 D-4→D-2): 수수료 $Z, 티어 W
```

**데이터 모델 (제안)**:
```
study_broker_product_rates
├─ broker_id
├─ product_id  (SKU)
├─ negotiated_broker_share  (협상된 수수료)
├─ negotiated_tier_id        (수기 매핑된 티어)
└─ effective_from, notes
```

## 4. 티어 — 자동화 X, 어드민 수기

- 어드민에서 **티어 구간·금액 설정** (TierConfig)
- **브로커별 티어를 어드민에서 수기 매핑**
- 자동화는 브로커 수백명 규모 도달 후 검토
- PRD §3.3 의 자동 카운트(Bronze/Silver/Gold/Platinum) 표는 **참고용**, 실제 운영은 수기

## 5. 같은 Entity, SKU 따라 역할 다름

베트남 유학원의 두 얼굴:
- SKU-1 (베트남→한국) 에서는 → **유학원** (자체 학생 관리, B2B, MVP 제외)
- SKU-2 (한국 D-4→D-2) 에서는 → **브로커** (자기네 졸업생을 우리에게 소개)

→ 모두 `study_brokers` 로 통합. broker × SKU 매트릭스가 자연스럽게 처리.

## 6. SKU 전환 — B→A 케이스만, 어드민 할인 설정

- SKU-1 의 어학당 옵션으로 시작 → 어학당 졸업 → 한국 거주 → SKU-2 본과 전과
- **2회 결제** (별도 application)
- 2회차 결제 시 **할인 적용** — 어드민에서 할인액 설정
- 다른 전환 케이스 없음 (단방향)

**데이터 모델 추가**:
```
study_products
└─ existing_customer_discount  (SKU-2 만 사용, 어드민 설정)
                                자격: 동일 학생이 SKU-1 어학당 옵션 완료
```

## 8. 영수증 발행

- 학생 결제 = **글로케어 단독 발행** (무조건)
- 브로커는 어필리에이트 수수료 수령 입장 (별도)

## 9. 베트남 송금 (어필리에이트 정산)

- 권장 수단: **Wise** (1차) 또는 **Payoneer** (2차)
- 한국 외환: 1회 $10K / 연 누계 $50K 이하 증빙 없이. 초과 시 계약서 + 인보이스 필수
- 베트남 측 원천징수 의무 없음 (베트남 거주자 본인 신고)
- 한국 법인 회계: 지급수수료 / 외주용역비 처리
- → 변호사·세무사 자문 시 **한-베 조세조약** 함께 검토

**어드민 정산 워크플로 (제안)**:
```
월말 → 어드민이 브로커별 정산 금액 합산
     → 인보이스 자동 생성 (PDF)
     → 브로커가 어드민에서 확인·승인
     → Wise 송금 → 송금 증빙 어드민에 첨부
     → 정산 완료 처리
```

브로커가 시스템에서 입력할 것:
- 수신 계좌 (은행명·계좌번호·SWIFT 코드 또는 Wise/Payoneer 이메일)
- 베트남 사업자등록번호 (개인사업자/법인)
