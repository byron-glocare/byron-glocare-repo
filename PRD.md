## 0. 빠른 요약 (TL;DR)

- **무엇**: 베트남 → 한국 유학생 모집/관리 플랫폼 "글로케어"
- **누가 쓰는가**: 학생(고객), 브로커(어필리에이트 파트너), 유학원(B2B 파트너), 글로케어 운영자
- **핵심 비즈니스**: 학생 1명 유치 → 정가 $2,500 기준, 채널에 따라 3가지 수익 구조
- **플랫폼 형태**: 관리자 웹 + 고객 웹/앱
- **개발 우선순위**: 1) 학생 가입·결제 + 어필리에이트 코드 트래킹, 2) 운영자 대시보드, 3) 브로커·유학원 포털, 4) 대학 접수 워크플로우

## 1. 용어 정의 (Glossary)

| 용어 | 정의 |
|---|---|
| **글로케어** | 플랫폼 운영사 |
| **학생** | 한국 유학을 희망하는 베트남 학생 (최종 고객) |
| **브로커 (Broker)** | 학생을 글로케어에 소개하는 어필리에이트 파트너. 일반 소개 브로커, 어학원 등 |
| **유학원 (Agency)** | 자체 유학 상담·관리 기능이 있고, 글로케어에는 대학 접수만 위탁하는 B2B 파트너 |
| **기본 상품** | $2,500 정가의 유학 패키지 (회사 기본 매출 $1,500 + 부가서비스 최대 $1,000) |
| **부가서비스** | 비자 대행, 기숙사 알선, 어학원 등록 등 선택 가능한 추가 상품 (최대 $1,000) <!-- TBD: 실제 부가서비스 항목 리스트 확정 필요 --> |

| **대학 소개비** | 학생이 대학에 등록 완료 시 대학에서 글로케어에 지급하는 수수료. 30~60만원 (대학마다 다름). 모든 채널 동일 |
| **브로커 수수료** | 브로커가 학생 1명당 가져가는 금액. $0~$1,000 (사전 협상) |
| **티어 인센티브** | 누적 소개 실적에 따라 회사가 브로커에게 추가 지급하는 보너스 ($0~$100) |
| **어필리에이트 코드** | 브로커별 고유 코드. 학생 가입 시 입력하면 그 브로커에게 수수료 귀속 |

## 2. 사용자 페르소나

### 2.1 학생 (Student)
- **언어**: 베트남어 (한국어·영어 일부)
- **사용 환경**: 모바일 우선 (베트남은 모바일 비중 높음)
- **목표**: 한국 대학 정보 확인, 유학 신청, 진행 상황 추적
- **이용 채널**: 1) 직접 방문(웹/앱), 2) 브로커 소개 링크, 3) 유학원에서 안내받음

### 2.2 브로커 (Broker)
- **누구**: 일반 소개인, 어학원, 한국어 학원, 베트남 현지 에이전트
- **목표**: 자기가 소개한 학생 추적, 수수료 확인, 수입 극대화
- **사용 환경**: 모바일/PC 혼용
- **핵심 니즈**: 자기 어필리에이트 코드 공유, 학생 진행 상황 가시화, 정산 투명성

### 2.3 유학원 (Agency)
- **누구**: 베트남 현지 유학원 (자체 학생 관리 기능 보유)
- **목표**: 자기 학생을 한국 대학에 빠르게 접수
- **사용 환경**: PC 위주
- **핵심 니즈**: 학생 정보 일괄 등록, 접수 진행 상태 확인, 대학별 요구 서류 가이드

### 2.4 글로케어 운영자 (Admin)
- **역할**: 채널 관리, 정산 처리, 대학 접수 처리, 학생 상담
- **사용 환경**: PC 위주, 다중 사용자(상담·접수·정산 등 역할 분리)
- **핵심 니즈**: 모든 학생·브로커·유학원·대학 데이터 통합 관리, 정산 자동 계산

## 3. 비즈니스 모델 (Business Rules)

> **시각적 참조**: `Glocare_MoHinhKinhDoanh.html` 파일 참조. 인터랙티브 시뮬레이터로 모든 케이스 확인 가능.

### 3.1 가격 구조 (공통)

| 항목 | 금액 |
|---|---|
| 정가 (List Price) | $2,500 (부가서비스 포함 최대치) |
| 회사 기본 매출 (Company Base) | $1,500 (고정) |
| 부가서비스 최대 | $1,000 |
| 대학 소개비 | 30~60만원 (대학별 상이, 모든 채널 동일) |
| 환율 (참고) | $1 = ₩1,400 = 26,000 ₫ |

### 3.2 채널 1: 학생 직접 방문

```
학생 문의 → 직접 상담 → 결제 → 유학 서비스 → 대학 등록
```

- **학생 결제**: $1,500 ~ $2,500 (기본 + 선택 부가서비스)
- **회사 수익**: 학생 결제 전액 + 대학 소개비 30~60만원
- **상시 40% 할인 적용** (정가 $2,500 → $1,500)

### 3.3 채널 2: 브로커 소개 (어필리에이트)

```
사전: 브로커 수수료 협상
  ↓
학생 가입 시 브로커 코드 입력
  ↓
결제 → 유학 서비스 → 대학 등록
```

**가격 결정 로직**:
- 회사 기본 매출 $1,500은 고정 (절대 깎이지 않음)
- 브로커가 가져갈 수수료(0~$1,000) = 학생 할인 조정 변수
- 브로커가 $0 가져가면 → 학생은 40% 할인 ($1,500 결제)
- 브로커가 $1,000 가져가면 → 학생은 정가 결제 ($2,500)
- **공식**: `학생 결제 = $1,500 + 브로커 수수료`, `학생 할인율 = (정가 - 학생결제) / 정가`

**티어 인센티브** (회사가 자기 마진에서 브로커에게 추가 지급):

| 티어 | 누적 소개 건수 (예시) | 추가 인센티브 |
|---|---|---|
| 🥉 Bronze | 1~5건 | $0 |
| 🥈 Silver | 6~20건 | +$50 |
| 🥇 Gold | 21~50건 | +$80 |
| 💎 Platinum | 51건 이상 | +$100 |

**부가서비스 수익**: 학생이 부가서비스 추가 시 전액 회사 매출 (브로커 영향 없음)

**대학 소개비**: 채널 1과 동일하게 글로케어가 수령

### 3.4 채널 3: 유학원 등록 학생

```
유학원 등록 학생 → 유학원 직접 상담·관리 → 글로케어에 접수 대행 의뢰 → 대학 접수
```

**역할 분담**:
- **유학원**: 학생 모집, 상담, 관리, 송영 등 유학원의 모든 역할 직접 수행
- **글로케어**: 대학 접수 대행만 수행
- **대학**: 접수 완료 시 글로케어에 소개비 30~60만원 지급

**정산**:
- 학생 ↔ 글로케어: 정산 없음
- 유학원 ↔ 글로케어: 정산 없음
- 대학 → 글로케어: 30~60만원 (회사 유일 수익)

## 4. 핵심 사용자 여정 (User Journey)

### 4.1 학생 - 채널 1 (직접)
1. 글로케어 웹/앱 접속
2. 대학·전공 검색·비교
3. 회원가입 (베트남어 UI)
4. 유학 상담 신청 (채팅 또는 예약)
5. 기본 상품 + 부가서비스 선택
6. 결제 ($1,500 ~ $2,500)
7. 마이페이지에서 진행 상태 추적
8. 서류 준비 가이드 수령·제출
9. 대학 합격·등록 완료
10. 한국 출국·정착 지원

### 4.2 학생 - 채널 2 (브로커 소개)
1. 브로커로부터 글로케어 추천 (URL + 코드)
2. 가입 단계에서 **브로커 코드 자동 입력** (URL 파라미터) 또는 **수동 입력**
3. 이후 채널 1과 동일하나 가격이 다르게 표시됨
4. 브로커는 학생 진행 상태를 자기 대시보드에서 확인

### 4.3 학생 - 채널 3 (유학원 경유)
1. 학생은 글로케어와 직접 접촉하지 않음
2. 유학원에서 상담·결제·관리 모두 수행
3. 유학원이 글로케어에 대학 접수만 위탁
4. 학생은 글로케어 시스템에 등록되지만, 로그인은 안 함 (조회만) <!-- TBD: 학생 본인 로그인 허용 여부 -->

### 4.4 브로커
1. 회원가입 → 글로케어 운영자 승인 후 활성화
2. 본인의 어필리에이트 코드·공유 링크 발급
3. 수수료 협상 (운영자와 1:1, 시스템 외 협상) <!-- TBD: 시스템 내 협상 도구 필요 여부 -->
4. 학생 소개 → 가입·결제 트래킹
5. 자기 학생 진행 상황 대시보드 확인
6. 누적 실적·티어 확인
7. 월별 정산 내역 확인·승인

### 4.5 유학원
1. 회원가입 → 글로케어 운영자 승인
2. 자기 학생 정보 등록 (개별 또는 엑셀 일괄)
3. 대학·학과 선택 → 접수 대행 의뢰
4. 진행 상태 확인 (서류 요청, 합격, 등록)
5. 대학 등록 완료 시 글로케어가 대학에서 소개비 수령 (유학원과 무관)

### 4.6 글로케어 운영자
1. 학생/브로커/유학원 신규 가입 승인
2. 학생별 진행 상태 관리 (상담 → 결제 → 서류 → 대학 접수 → 합격 → 등록)
3. 대학별 요구 서류·일정 관리
4. 결제·환불 처리
5. 브로커 정산 (월별, 티어 자동 계산)
6. 대학 소개비 수금 확인
7. 보고서·분석 (채널별 수익, 브로커 실적, 대학 합격률 등)

## 5. 기능 요구사항 (Features)

> 화면 구조·메뉴 구성은 Claude Code가 이 기능 리스트를 기반으로 IA를 설계.

### 5.1 학생 - 고객 웹/앱

**인증·계정**
- 회원가입 (이메일/SNS), 로그인, 비밀번호 재설정
- 베트남어 우선, 한국어/영어 선택 가능
- 브로커 코드 입력 (URL 자동 입력 또는 수동)

**탐색**
- 한국 대학 목록·검색·필터 (지역·전공·학비·언어 등)
- 대학별 상세 페이지 (학과·입학 요건·기숙사·학비)
- 학과 비교

**상담·신청**
- 유학 상담 신청 (채팅 또는 예약) <!-- TBD: 실시간 채팅 vs 폼 기반 vs 외부 메신저 연동 -->
- 기본 상품 + 부가서비스 선택
- 견적 확인 (브로커 코드 적용 시 자동 할인 반영)
- 결제 (베트남 결제 수단 + 카드) <!-- TBD: 베트남 현지 결제수단 - MoMo, VNPay, ZaloPay 등 -->

**마이페이지**
- 진행 상태 (상담 → 결제 → 서류 준비 → 접수 → 합격 → 등록)
- 서류 업로드·요청 확인
- 알림 (신규 메시지, 상태 변경, 합격 통지 등)
- 영수증·계약서 다운로드 <!-- TBD: 영수증 발행 주체 - 글로케어 단독 vs 브로커 명의 -->

### 5.2 브로커 - 파트너 웹

**가입·인증**
- 신청 → 운영자 승인
- 본인 어필리에이트 코드 발급
- 공유 링크 생성 (소셜·메신저용)

**대시보드**
- 본인 누적 실적 (소개 건수, 결제 완료 건수, 수익)
- 현재 티어 + 다음 티어까지 남은 건수
- 월별 추이 그래프

**학생 관리**
- 본인 코드로 가입한 학생 목록
- 학생별 진행 상태 (이름은 마스킹 여부 결정 필요) <!-- TBD: 개인정보 보호 정책 -->
- 결제 완료 학생만 수수료 발생

**정산**
- 월별 정산 내역
- 수수료 = 기본 수수료 + 티어 인센티브
- 정산 승인·요청 <!-- TBD: 정산 주기 - 매월? 매주? 등록 완료 시? -->

### 5.3 유학원 - 파트너 웹

**가입·인증**
- 사업자 등록증 등 서류 제출 → 운영자 승인

**학생 관리**
- 학생 정보 등록 (개별/엑셀 일괄)
- 대학·학과 선택 후 접수 대행 의뢰
- 진행 상태 확인 (서류 → 접수 → 합격 → 등록)

**서류·소통**
- 요구 서류 가이드 (대학별)
- 운영자와 메시지 <!-- TBD: 채팅 vs 티켓 vs 메일 -->

### 5.4 운영자 - 관리자 웹

## 6. 데이터 모델 개요

> 상세 ERD는 Claude Code가 이 명세를 기반으로 설계. 핵심 엔티티만 나열.

**핵심 엔티티**:
- `User` (베이스: 학생/브로커/유학원/운영자 공통)
- `Student` (User 확장 또는 별도)
- `Broker` (User 확장, 어필리에이트 코드 보유)
- `Agency` (User 확장)
- `Admin` (User 확장, 권한 관리)
- `University` (대학 + 학과)
- `Major` (학과·전공·입학 요건)
- `Product` (기본 상품)
- `AddOn` (부가서비스)
- `Application` (학생의 유학 신청 - 진행 상태 관리)
- `Payment` (결제 내역)
- `BrokerCommission` (브로커 정산 내역)
- `TierConfig` (티어 정책 - 운영자 변경 가능)
- `UniversityReferralFee` (대학 소개비 수금)
- `Document` (서류 - 학생 업로드)

**핵심 관계**:
- `Student` ←(broker_code)→ `Broker` (어필리에이트 트래킹)
- `Student` ←(agency_id)→ `Agency` (유학원 등록 학생)
- `Application` ←→ `University`, `Major`
- `Application` → `Payment` → `BrokerCommission`

**진행 상태(Status) 머신** (학생 신청 기준):
```
INQUIRY → CONSULTATION → CONTRACT → PAYMENT_PENDING →
PAID → DOCUMENT_PREP → SUBMITTED → ACCEPTED →
ENROLLED → COMPLETED
              ↓ (도중 취소)
           CANCELLED
```

## 7. 정산 로직 (구현 명세)

> Claude Code가 정산 모듈 구현 시 그대로 코드화 가능한 수준으로 명시.

### 7.1 학생 결제 시점

```
입력: 학생 결제 정보
  - product_price = $1,500 (회사 기본 매출, 고정)
  - addon_total = 학생이 선택한 부가서비스 합계 (0 ~ $1,000)
  - broker_code = 브로커 코드 (있을 수도, 없을 수도)
  - is_via_agency = 유학원 경로 여부

분기:
  - is_via_agency == true:
      학생 결제 = 0 (글로케어는 학생에게 청구 안 함)
      회사 매출 = 0 (이 시점)
      → 대학 소개비만 추후 수령
  
  - broker_code 있음:
      broker_share = Broker 테이블에서 사전 협상된 수수료 ($0~$1,000)
      학생 결제 = product_price + broker_share + addon_total
      회사 매출 = product_price + addon_total
      회사 부채 = broker_share (브로커에게 지급 예정)
  
  - broker_code 없음 (직접):
      학생 결제 = product_price + addon_total
      회사 매출 = product_price + addon_total
```

### 7.2 대학 등록 완료 시점

```
대학 소개비 30~60만원 수령 (대학별 사전 설정 금액)
→ 회사 매출에 추가
```

### 7.3 브로커 정산 시점 (월별 일괄)

```
1. 해당 월에 PAID 상태가 된 학생들 중 broker_code가 있는 건 수집
2. 각 학생별 broker_share 합산
3. 브로커별 누적 PAID 건수 확인 → 현재 티어 결정
4. 티어 인센티브 = 티어 보너스 × 해당 월 건수
5. 정산 = 학생별 broker_share 합 + 티어 인센티브
6. 회사 매출 차감: 티어 인센티브만큼 (broker_share는 처음부터 부채로 잡혀있음)
```

## Claude Code에게 작업 지침

1. **이 PRD의 비즈니스 규칙(3장·7장)은 절대 임의 변경하지 말 것.** 그것이 사업의 핵심.
2. **TBD 항목을 발견하면 코딩하기 전에 사용자에게 결정을 요청할 것.** 임의 가정 금지.
3. **Phase 1 MVP 범위를 먼저 완료할 것.** Phase 2 이상은 사용자 승인 후 진행.
4. **데이터 모델은 사용자 확인 후 마이그레이션 작성.**
5. **다국어 키는 처음부터 분리.** 하드코딩된 베트남어/한국어 금지.
6. **테스트**: 정산 로직(7장)은 단위 테스트 필수. 채널별 시나리오 케이스 커버.
7. **개인정보**: 학생 여권·학적·재정 증명서 등은 암호화·접근 제한 처음부터 적용.
